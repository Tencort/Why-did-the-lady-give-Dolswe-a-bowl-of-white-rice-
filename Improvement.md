## [2026-03-17] #001 — 단일 페이지 + 지도 중심 UI로 구조 개편

### 제안자
사용자

### 현재 구조
- `/` (랜딩) → `/recommend` (추천) 의 2페이지 구조
- 랜딩: 위치 허용 → 조건 설정
- 추천: 대화형 턴제 → 지도 탐색 (순차 전환)

### 문제점
- 페이지 이동 시 URL 파라미터(lat/lng/headcount/categories)로 상태 전달 — 취약
- 지도가 대화형 모드 이후에만 등장하는 보조 요소로 취급됨
- 조건 설정과 추천이 분리되어 흐름이 끊김

### 개선 방향
- **단일 페이지** (`/`)로 통합
- **지도를 메인 화면으로** 항상 배경에 표시
- **인앱 메시지(Bottom Sheet / 오버레이 카드)** 형식으로 마님 대사 + 수락/거절 버튼 표시
  - 대화 중: 하단 고정 카드에 마님 대사 + 음식점 정보 + 버튼
  - 지도 탐색 중: 카드 최소화, 핀 클릭 시 카드 팝업
  - 수락 완료: 선택된 핀 강조 + 카드에 최종 결과 표시
- 조건 설정(인원/카테고리)은 상단 바 또는 첫 진입 시 오버레이로 처리

### 기대 효과
- 지도 → 대사 → 선택의 맥락이 한 화면에서 연결됨
- 모바일 UX 자연스러움 (카카오맵/배민 앱 패턴과 유사)
- URL 파라미터 의존 제거 → 상태 안정성 향상

### 상태
`검토 중`

## [2026-03-17] #002 — 위치 좌표 전달 버그 + API 에러 무시 버그

### 제안자
사용자 (황무지 오류 리포트)

### 버그 내용

**버그 A — null 좌표가 "null" 문자열로 전달됨**
- `String(null)` = `"null"` (문자열)
- `parseFloat("null")` = `NaN` (0이 아님, `|| '0'` fallback 미작동)
- 결과: 좌표 없이 API 호출 → 0 결과 → "황무지" 메시지

**버그 B — 카카오 API 실패 시 조용히 무시**
- REST API 키 오류 등 `!response.ok` 발생 시 `continue`로 넘어감
- 모든 카테고리 실패해도 "결과 없음"으로 처리되어 실제 원인 불명

### 수정 내용
- `page.tsx`: `!geo.lat || !geo.lng` 가드 추가
- `recommend/route.ts`: 좌표 NaN/0 유효성 검사, 전체 API 실패 시 502 에러 반환
- 근본 원인: `.env.local`에 플레이스홀더 값이 그대로 남아있어 REST API 키 미설정

### 상태
`수정 완료`
