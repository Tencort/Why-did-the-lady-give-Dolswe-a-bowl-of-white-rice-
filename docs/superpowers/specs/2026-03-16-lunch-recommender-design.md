# 왜 마님은 돌쇠에게 쌀밥을 주었을까 — 서비스 설계 문서

## 개요

직장인 점심 추천 서비스. 사극 세계관(마님/돌쇠)을 통해 메뉴 선택에 재미를 부여한다. 기존 점메추의 단순 랜덤 추천과 달리, 마님이 추천 → 돌쇠가 수락/거절하는 턴제 인터랙션과 거절 시 페널티 시스템을 통해 게이미피케이션 요소를 더한다.

## 핵심 컨셉

- **마님**: 점심 메뉴를 추천하는 존재 (MVP에서는 AI/시스템, 이후 팀 모드에서 실제 상사)
- **돌쇠**: 사용자. 마님의 추천을 받아들이거나 거절할 수 있다
- **쌀밥**: 거절 가능 횟수를 나타내는 자원 (3개). 다 쓰면 직접 골라야 한다

## MVP 범위

- 1인 모드 (AI 마님)만 구현
- 로그인 없음 (위치 허용만)
- 재미 인터랙션 포함 (대사 시스템, 거절 페널티)
- 팀 모드는 이후 확장

## 기술 스택

- **프론트엔드**: React + Next.js
- **지도**: 카카오맵 SDK
- **음식점 데이터**: 카카오맵 장소 검색 API (카테고리 + 거리 기반)
- **대사 시스템**: 사전 정의 대사 풀 (MVP)
- **상태 관리**: 클라이언트 상태만 (DB 없음, 새로고침 시 리셋)
- **캐릭터**: 사극풍 도트/픽셀아트 (정적 오버레이)
- **환경변수**: 카카오맵 API 키는 `.env.local`로 관리

## 유저 플로우

```
[접속] → [위치 허용 요청]
  → 허용 → [조건 설정 (인원/카테고리)]
  → 거부 → [수동 주소 입력 fallback]
    → [마님 등장 + 추천 (대화형 턴제)]
      → 수락 → [음식점 상세 + 지도 위치 표시] → 끝
      → 거절 → [돌쇠 변명 + 마님 반응]
        → 재추천 (마님이 다른 메뉴 제시, 이전 거절 음식점 제외)
        → 3회 거절 → [지도 탐색 모드 전환: "네가 직접 골라라"]
          → 핀 선택 시 마님 한마디 → 확정
          → 결정 못할 시 → "쌀밥을 주시옵소서" → 마님이 강제 선택 ("쌀밥" 이벤트)
```

## 화면 구성

### 1. 조건 설정 화면

- 인원 선택 (1~10명) — 대사 연출에 활용 ("혼밥이냐?", "여럿이 가느냐")
- 카테고리 선택 (한식/중식/일식/양식/분식 등 멀티 선택)
- "마님께 여쭙기" 버튼으로 추천 시작
- 사극풍 다크 테마, 금색(#ffd700) 포인트 컬러

### 2. 대화형 추천 화면 (턴제)

- 마님 말풍선: 추천 대사 + 음식점 정보 (이름, 카테고리, 거리)
- 수락 버튼: "감사히 받겠습니다" (녹색 계열)
- 거절 버튼: "사양하겠습니다" (적색 계열)
- 하단 쌀밥 잔여량 게이지 (🍚🍚🍚 → 거절마다 하나씩 사라짐)
- 거절 시: 돌쇠 변명 → 마님 반응 → 다음 추천 순으로 진행

### 3. 지도 탐색 모드 (메인)

- 3회 거절 시 전환 ("쌀밥 박탈" 경고 표시)
- 카카오맵 위에 픽셀아트 마님/돌쇠 캐릭터 정적 오버레이
- 음식점 핀 표시, 핀 선택 시 마님 코멘트
- 상단에 "쌀밥 박탈" 상태 표시

## 아키텍처

```
[프론트엔드 - Next.js]
  ├── 페이지
  │   ├── / (랜딩 → 위치 허용 → 조건 설정)
  │   └── /recommend (대화형 추천 + 지도 탐색)
  │
  ├── 주요 컴포넌트
  │   ├── ConditionForm (인원/카테고리)
  │   ├── DialogueBox (마님/돌쇠 대화창 - RPG 턴제)
  │   ├── MapView (카카오맵 + 캐릭터 오버레이)
  │   ├── RiceBowlGauge (쌀밥 잔여량 표시)
  │   └── PixelCharacter (마님/돌쇠 픽셀아트 정적 이미지)
  │
  └── 상태 관리
      ├── 거절 횟수 (0~3)
      ├── 현재 모드 (dialogue | map)
      ├── 추천 음식점 목록
      └── 거절한 음식점 ID 목록 (중복 추천 방지)

[백엔드 - Next.js API Routes]
  ├── /api/recommend (조건 기반 음식점 추천)
  │   └── 카카오맵 API → 반경 1km 내 카테고리 기반 장소 검색
  │       → 거절한 음식점 제외 후 랜덤 선택
  ├── /api/dialogue (마님/돌쇠 대사 생성)
  │   └── 사전 정의 대사 풀에서 상황별 랜덤 선택
  └── /api/comment (지도 모드에서 마님 코멘트)
      └── 음식점 카테고리/거리 기반 조건부 대사

[외부 API]
  └── 카카오맵 API (장소 검색 + 지도 렌더링)
```

### 추천 알고리즘 (MVP)

1. 카카오맵 API로 현재 위치 반경 1km 내 음식점 검색 (카테고리 필터 적용)
2. 이미 거절한 음식점 ID 제외
3. 남은 목록에서 랜덤 선택
4. 검색 결과 0건 시: 마님 대사 "이 근방에는 먹을 곳이 없구나... 반경을 넓혀보겠느냐?"
5. 추천 가능 음식점 < 3개 시: 남은 수만큼만 추천 후 바로 지도 모드 전환

## 대사 시스템

### 마님 추천 대사 (첫 등장)

- "오늘은 {카테고리}가 딱이니라. {음식점}에 가거라!"
- "내 입맛은 틀린 적이 없느니라. {음식점}을 먹어라!"
- "돌쇠야, 오늘 점심은 내가 정했다. {음식점}이다!"

### 거절 시 돌쇠 변명 (랜덤)

- "마님, 어제도 그걸 먹었사옵니다..."
- "소인, 요즘 몸이 안 좋아 기름진 건 좀..."
- "마님 은혜는 감사하오나, 지갑 사정이..."

### 마님 반응 (단계별)

| 거절 횟수 | 마님 감정 | 예시 대사 |
|-----------|----------|----------|
| 1회 | 서운 | "허, 내 안목을 의심하느냐..." |
| 2회 | 화남 | "이놈이 감히! 두 번이나 거절이냐!" |
| 3회 | 폭발 | "좋다! 네 놈이 직접 골라봐라. 쌀밥은 없다!" |

### 지도 모드 마님 코멘트 (핀 선택 시)

- 가까운 곳 (200m 이내): "가까운 곳만 찾다니, 게으른 놈..."
- 먼 곳 (800m 이상): "거기까지 걸어갈 성의는 있느냐?"
- 마님 추천과 같은 카테고리: "...결국 내 말이 맞지 않더냐"
- 그 외: "흥, 네 놈 알아서 해라"

### 인원 기반 대사

- 1명: "혼밥이냐? 쓸쓸한 놈..."
- 2~3명: "소수정예로 가는구나"
- 4명 이상: "대식구로구나, 넓은 곳을 찾아야겠다"

## 에러 처리

| 상황 | 처리 |
|------|------|
| 위치 허용 거부 | 수동 주소 입력 폼 표시 |
| 검색 결과 0건 | 마님 대사로 안내 + 반경 확대 제안 |
| 추천 가능 음식점 < 3개 | 남은 수만큼 추천 후 지도 모드 전환 |
| 카카오맵 API 장애 | "마님이 잠시 자리를 비우셨습니다" + 재시도 버튼 |
| 새로고침 시 | 상태 리셋 (별도 경고 없이 처음부터 시작) |

### 쌀밥 이벤트 (지도 모드에서 결정 못할 때)

마님이 다시 나서서 강제로 음식점을 골라주는 이벤트. 돌쇠가 "쌀밥을 주시옵소서" 버튼을 누르면 발동.

- "에잇, 이 불쌍한 놈! 내가 정해주마. 쌀밥이다!"
- "결정도 못하느냐... 가엾구나. 내가 골라주겠다!"
- "보다 못해 나서마. 이리 오너라, 쌀밥을 주겠다!"

## 디자인 톤

- **테마**: 사극풍 다크 (#1a1a2e 배경, #ffd700 금색 포인트)
- **캐릭터**: 사극풍 도트/픽셀아트 (정적 이미지) — "아이탈출" 게임의 캐릭터 디자인 참고 (둥글둥글한 머리, 심플한 표정, 2~3등신 비율)
- **폰트**: 나눔명조 (Google Fonts, 무료)
- **버튼/UI 텍스트**: 사극 말투 사용 ("여쭙기", "감사히 받겠습니다", "사양하겠습니다")

## 이후 확장 (MVP 이후)

- 팀 모드: 마님(상사)이 실제로 메뉴 추천 → 돌쇠(팀원)들이 반응
- 소셜 로그인 + 히스토리 저장
- AI 대사 생성 (GPT/Claude 연동)
- 캐릭터 애니메이션 (지도 위 이동)
- 날씨/요일/시즌 기반 AI 맥락 추천
- 별점/가격대 데이터 연동 (네이버 API 또는 자체 DB)
